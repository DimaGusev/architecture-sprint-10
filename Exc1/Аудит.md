
# Cписок проблемных зон

- для регистрации или записи клиента, работник ресепшена должен скаживать файл со всеми данными всех клиентов, что избыточно, потому что работнику нужно только добавить туда строчку, но это не возможно без скачивания всего файла. После скачивания файла, создаётся копия данных на локальном компьютере, данные утекают из единого хранилища.
- работники ресепшена имеют доступ ко всем данным всех клиентов.
- с клиентом не оформляется информированное согласие на обработку персональных данных
- нет разграничения доступа на файлы с клиентами, медицинскими картами, журналы записей и анализов
- врачу доступно редактирование записей других врачей, а так же доступны персональные данных всех пациентов
- врачу доступны медицинские карты всех клиентов, даже тех, кого он не принимает
- врачу доступны анализы всех клиентов, даже тех, кого он не принимает
- приложения 1С работают в файловом режиме, что ограничевает масштабирование, а так же создают риски безопасности, убитывая что файл хранится вместе с остальными данными
- не защищённый протокол обмена данными межды 1С компонентами
- дублирование финансовой информации в Excel, создаёт риски утечки данных
- аналитику доступны все персональный данные, включая специальный данные, хотя для работы они скорее всего не нужны, достаточно обезличенных данных
- все операции выполняются вручную, повышая вероятность ошибки и создавая риски безопасности
- результаты работы с данными сохраняются и читаются в файлах и IT-системах без аудита действий
- Внутренние потоки данных между IT-продуктами никак не контролируются

# Cписок данных для защиты

Среди данных для защиты можно выделить следующие категории и способы защиты:

| Категория данных    | Примеры данных | Способ защиты |
| -------- | ------- | ------- |
| Персональные данные (PII)  | ФИО, адрес, места работы учёбы, телефон, почта и т.д    | шифрование при хранении и передаче, маскирование и обезличивание при обработке, RBAC при обработке |
| Специальные данные (PHI) | данные о здоровье, включая заболевания, диагнозы и анализы.     | шифрование при хранении и передаче, RBAC при обработке |
| Финансовые данные    | финансовые данные    | шифрование при хранении и передаче, RBAC при обработке |
| Данные о персонале    | данные о персонале    | шифрование при хранении и передаче, RBAC при обработке |
| Информация о заказе оборудования    |  данные об оборудовании    | шифрование при передаче, RBAC при обработке |
| Журнал приёма данных    |  данные о записях на приём клиентов    | маскирование(от замены части данных, вплоть до полного удалении данных, оставляя только факт наличии зависи на данное время), RBAC при обработке |


# Тегирования данных

Тегировать данные можно на основании категорий данных выделенный выше тегами PII, PHI, finance_secret, staff_secret и facilities_secret.

Выбор инструмента для тегирования сложен. Практически все предложенные в теоретическом курсе инструменты являются коммерческими, большинство не работают с Россией из за санкций, проведение оплаты является сложным действием, а самое главное присутствуют гигантские риски. Единственный open source инструмент из предложенных - Apache Atlas, он и предполагается для использования.

# Обеспечение конфиденциальности данных

- Конфиденциальность данных должна обеспечиваться за счёт шифрования (AES-256) такий данных при хранении и передаче (TLS 1.2+). Также за счёт маскирования и обезличивания при обработке. 

- PII данные должны хранится в одной базе, в других база только идентификаторы пользователей. 

- Доступ к данным должен контролировать через RBAC, а к конкретным данным может ограничиваться через ABAC. 

- Должен применяться принцип минимизации данных. Человек должен иметь доступ только к тем данным, которые нужны для работы. 

- Работа 1С должна производится через клиент-серверный подход с включенным режимом обработки персональных данных, межплатформенный обмен должен происходит по защищённым протоколам. 

- Операции с данными должны проводиться через специально спроектированны ИТ системы, а не вручную. 

- В качестве инструмента аутентификации и авторизации - KeyCloak. 

- Внедрить аудит действий с данными.
